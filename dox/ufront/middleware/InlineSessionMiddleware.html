class ufront.middleware.InlineSessionMiddleware implements UFMiddleware

Available on all platforms

Make sure we init() sessions before the request starts, and commit() them before it ends

@author Jason O'Neil

Class Fields

static var alwaysStart:Bool

Should we start a session for every request, or only if one already exists? If false, one will only be started if init() is called specifically on one request. (For example, when they log in). From there onwards it will initialize with each request.

Instance Fields

function new():Void

function requestIn(ctx:HttpContext):Surprise<Noise, Error>

Start the session if a SessionID exists in the request, or if alwaysStart is true.

function responseOut(ctx:HttpContext):Surprise<Noise, Error>

If the session is active, commit the session before finishing the request.
